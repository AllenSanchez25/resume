# 技术文档新手避坑指南：如何用 Markdown 排版一份合格的说明文档》

作为技术文档新手，我们常常会陷入“只会写文章，不会排版”的尴尬境地。Markdown 虽然上手简单，但规范的排版直接决定了文档是否专业、是否易读。本文总结五个最常见的“坑”，助你避免排版车祸现场。

## 一、常见的五大“坑”
### 🕳️ 坑1：标题层级混乱
错误示范：
# 第一章 介绍
## 1.1 背景
### 1.1.1 当前环境

#### 避坑指南： Markdown 的层级不应由编号决定，而应由内容逻辑决定。标题后面不要加“#”占位符。推荐遵循 “层级递进，不多不跳” 原则（即不要出现 H1 之后直接跳 H3 的情况）。
---
### 🕳️ 坑2：不分段或过度换行
错误示范：
这是第一句话。
这是第二句话。  

#### **避坑指南**： Markdown 中，为了换行需要加两个空格后回车，或者直接留一个空行来分段。用空行分段会让读者视觉更清晰。
---
### 🕳️ 坑3：滥用加粗(**)斜体(*)
错误示范： 整句话都是 **这是一个非常重要的通知，大家一定要注意。**
#### **避坑指南**： 加粗只用于强调关键词（如警告、文件名、按钮名称）。斜体用于术语或书名。滥用会干扰阅读节奏。
---
### 🕳️ 坑4：列表缩进错误
错误示范：
1. 第一步
   - 子步骤
#### **避坑指南**： 在无序列表（-）或有序列表（1.）下缩进子内容，至少要敲4个空格才能产生次层级效果。
---
### 🕳️ 坑5：代码块没有指定语言
错误示范 ：
```
print
```
("Hello World")
```
text
```
#### **避坑指南**： 在三个反引号后加上语言名（如 python, bash, json），能让代码高亮，极大提升专业度。
---
## 二、一份合格的 Markdown 文档长什么样？
以下是一个“合格说明书”的模板：

### 1. 标题与目录
```
# 软件安装与配置指南

## 目录
1. [环境要求](#环境要求)
2. [安装步骤](#安装步骤)
3. [常见问题](#常见问题)
```
### 2. 规范的表格展示
```
| 参数名 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `timeout` | 60s | 请求超时时间 |
| `retry` | 3 | 失败重试次数 |
```
### 3. 明显的警示块（引文或 HTML）
```
> **⚠️ 警告：** 执行此操作前请务必备份数据库！
```
#### 4. 清晰的结构化图片引用
```
![架构图](./assets/architecture.png)
_图1：系统整体架构图_
```
---
## 三、结语
写好 Markdown 不是炫技，而是让读者快速获取信息。哪怕你不懂复杂的 HTML 和 CSS，只要遵循“层级清晰、排版合理、表意准确”这三条铁律，你的文档就已经超过了 80% 的新手。
